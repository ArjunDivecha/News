## The catch
There's a known bug ([#21974](https://github.com/anthropics/claude-code/issues/21974)): when you resume a session from the Agent View list (Enter/right-arrow to attach, or `/continue`), Claude Code spawns a fresh process and **drops** `--dangerously-skip-permissions`.

## Workarounds

**1. Launch the agent process itself with the flag, then attach:**
```bash
claude --dangerously-skip-permissions agents
```
Sessions you attach to from inside that view inherit the flag.

**2. For background jobs, set it at launch:**
```bash
claude --dangerously-skip-permissions --bg "your prompt"
```
Then `claude agents` to monitor/attach.
